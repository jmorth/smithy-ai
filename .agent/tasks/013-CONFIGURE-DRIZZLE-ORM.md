# Task 013: Configure Drizzle ORM

## Summary
Install Drizzle ORM, Drizzle Kit, and the PostgreSQL driver in `apps/api`. Create `drizzle.config.ts` for migration management, `db.ts` as the database client factory with connection pooling, and the schema barrel export. This establishes the data access layer that all API modules use to interact with PostgreSQL.

## Phase
Phase 1: Foundation & Infrastructure

## Dependencies
- **Depends on**: 007 (NestJS API App scaffold), 011 (Docker Compose Dev Stack — need running Postgres for testing)
- **Blocks**: 014 (Packages Schema), 015 (Workers Schema), 016 (Workflows Schema), 017 (Jobs/Notifications Schema), 018 (Relations and Migration)

## Architecture Reference
Drizzle ORM is the chosen data access layer for Smithy, selected for its type-safe schema definitions, SQL-like query builder, and zero-overhead approach. The configuration lives in `apps/api` since only the API application talks directly to PostgreSQL. The schema directory will house all table definitions, and Drizzle Kit manages migrations.

Key design decisions:
- Use `postgres` (postgres.js) as the PostgreSQL driver for best performance
- Connection pooling is handled by the driver
- Schema files are co-located in `src/database/schema/`
- Migrations are generated by Drizzle Kit and stored in `drizzle/` (or `src/database/migrations/`)

## Files and Folders
- `/apps/api/drizzle.config.ts` — Drizzle Kit configuration (schema path, migration output, database connection)
- `/apps/api/src/database/db.ts` — Database client factory exporting a configured Drizzle instance
- `/apps/api/src/database/schema/index.ts` — Barrel export for all schema definitions (initially empty, populated by tasks 014-017)

## Implementation Notes
- Install: `drizzle-orm`, `postgres` (the postgres.js package). Dev: `drizzle-kit`.
- The `drizzle.config.ts` should use `defineConfig` from `drizzle-kit`:
  ```ts
  import { defineConfig } from "drizzle-kit";
  export default defineConfig({
    schema: "./src/database/schema/index.ts",
    out: "./drizzle",
    dialect: "postgresql",
    dbCredentials: { url: process.env.DATABASE_URL! },
  });
  ```
- The `db.ts` client factory pattern:
  ```ts
  import { drizzle } from "drizzle-orm/postgres-js";
  import postgres from "postgres";
  import * as schema from "./schema";

  const client = postgres(process.env.DATABASE_URL!, { max: 10 });
  export const db = drizzle(client, { schema });
  ```
- Consider wrapping the client creation in a function for testability and NestJS dependency injection.
- The `db:generate` script should run `drizzle-kit generate`.
- The `db:migrate` script should run `drizzle-kit migrate`.
- The `db:studio` script should run `drizzle-kit studio`.
- Do NOT create actual schema tables yet — that is tasks 014-017.
